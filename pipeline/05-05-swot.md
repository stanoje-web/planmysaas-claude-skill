# Stage 5.5 — SWOT

Generate `./planmysaas-blueprint/05-05-swot.md`. Read `03-01-competitive-landscape.md`, `04-01-willingness-to-pay.md`, and `02-03-demand-signals.md` first — every item below must trace back to a specific finding in one of these files.

Unlike the old combined analysis file this replaces, this stage exists as its own trackable artifact — treat it as a complete, standalone strategic document, not a fragment. A reader should be able to open this file alone and understand the product's competitive position without needing the source files open beside it, even though every claim in it is grounded in them.

## Output structure

```markdown
# SWOT — <Project name>

> A traceable SWOT — every claim below is grounded in a specific, named finding from the research files, not general impression.

## Summary
2-3 sentences: the single strongest strength, the single most dangerous weakness or threat, and the net read on whether the product's position is defensible.

## How this connects to Porter's Five Forces
1 sentence noting that the Threats quadrant here should broadly align with the high-scoring forces in `05-06-porters-five-forces.md` once that stage is generated later in this same phase — this file stands on its own for now, grounded only in the three files listed above.

## Strengths
4-6 items. Each: the strength, then a 1-line reasoning that names the specific source file and finding it comes from (e.g. "per `04-01-willingness-to-pay.md`, the target persona's stated budget sits above every named competitor's entry tier").

## Weaknesses
4-6 items, same format — internal blockers (team gaps, product gaps, cost structure), each traced to a specific finding. Draw on gaps implied by `03-01-competitive-landscape.md` (where incumbents are strong and this product isn't yet) as much as internal ones.

## Opportunities
4-6 items, same format — external tailwinds (market timing, underserved segments, competitor blind spots), each traced to a specific finding from `03-01-competitive-landscape.md` or `02-03-demand-signals.md`.

## Threats
4-6 items, same format — substitutes, competitors, regulation, commoditization risk, each traced to a specific finding. At least one threat should come from `04-01-willingness-to-pay.md` if the pricing findings expose a fragility (e.g. a persona whose willingness-to-pay is marginal and price-sensitive).

## Quadrant cross-check
Before finalizing, check for items that actually belong in a different quadrant than where they were placed (a "weakness" that's really a mirror of a named "threat", or a "strength" that only holds if a named "opportunity" materializes). Note any such pairing explicitly — 1-2 sentences.

## Worked example of an evidence-traced item
Show one fully-worked example so the format is unambiguous, e.g.:
> **Weakness — No existing distribution.** Per `03-01-competitive-landscape.md`, all 3 named direct competitors have 3+ years of organic SEO content and an established affiliate network; this product has neither, which means customer acquisition cost will front-load into paid channels until organic traction builds.

Every other item in the four lists should read at this level of specificity — a named source, a named fact, and a clear implication.

## What this means strategically
1 paragraph: which quadrant dominates (is this a strengths-and-opportunities story, or a weaknesses-and-threats story), and the one move that would shift the balance most. Name the specific action, not a general direction.
```

## Quality rules

- Every single item across all four lists must carry a reasoning clause that names the source file and the specific finding — a strength, weakness, opportunity, or threat with no traceable evidence is rejected outright, not softened.
- Do not restate the same underlying fact across multiple items dressed differently — each item should surface a distinct signal.
- Items must be specific to this product and this competitive set — generic SWOT filler ("strong team", "market competition") without a citation is rejected.
- If a quadrant is thin on real evidence (e.g. only 2 traceable threats exist), report fewer items rather than inventing filler to hit the 4-6 range.
- The quadrant cross-check is mandatory even if it finds nothing — say "no cross-quadrant overlap found" rather than omitting the section.
- The strategic-implication paragraph must name one specific move, not a vague direction like "focus on strengths."

## Tone

Evidence-driven strategist. Every claim is a citation, not an opinion. Willing to say a strength is thin if the evidence for it is thin.
