## How to use

1. Read **01-01-problem-validation.md** first. If anything's wrong, fix it there — later stages build on it.
2. Skim Phases 2-4 for market, competitive, and economic context.
3. Read **05-11-the-roast.md** before you get too attached to the idea — it's deliberately adversarial.
4. Use **06-architecture.md** as your engineering north star and **07-features.md** in priority order (P0 → P3).
5. **Open 10-build-playbook.md.** Start with Build Step 01. Every step has an acceptance rubric — do not advance until every checkbox passes.
6. Use **09-release-plan.md** to set sprint goals and avoid scope creep.
7. Run the **dashboard** (`cd dashboard && npm run dev`) any time you want the visual view instead of reading markdown.
